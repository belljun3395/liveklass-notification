м •мұ…мқҙ м Ғмҡ©лҗҳлҠ” л°©мӢқ

`NotificationJob`мқҖ л‘җ к°Җм§Җ л°©мӢқмңјлЎң м •мұ…мқ„ мӮ¬мҡ©н•©лӢҲлӢӨ.

  1. Job мғҒнғң м „мқҙ — transitionTo(policy): `policy.fromStatuses`м—җ нҳ„мһ¬ мғҒнғңк°Җ нҸ¬н•Ёлҗҳл©ҙ `policy.toStatus`лЎң м „мқҙ

  2. Notification мғҒнғң м „мқҙ кІҖмҰқ — assertNotificationTransitionAllowed(policy, from, to): м •мұ…м—җ м„ м–ёлҗң NotificationTransition м§‘н•©м—җ н•ҙлӢ№ м „мқҙк°Җ нҸ¬н•ЁлҗҳлҠ”м§Җ нҷ•мқё

// Job мғҒнғң м „мқҙ (мҳҲ: markProcessing)
public boolean markProcessing() {
    return transitionTo(JobNotificationPolicy.TO_PROCESSING);
}

// Notification мғҒнғң м „мқҙ кІҖмҰқ (мҳҲ: startSendingNotification)
public void startSendingNotification(Notification n) {
    assertNotificationTransitionAllowed(
        JobNotificationPolicy.TO_PROCESSING,
        n.getStatus(), NotificationStatus.SENDING);
    n.markSending();
}

мӨ‘ліө л°ңмҶЎ л°©м§Җ л¶„м„қ

мқҙ мӢңмҠӨн…ңмқҳ н•өмӢ¬ мҡ”кө¬мӮ¬н•ӯмқҖ к°ҷмқҖ Notificationмқҙ л‘җ лІҲ л°ңмҶЎлҗҳл©ҙ м•Ҳ лҗңлӢӨлҠ” кІғмһ…лӢҲлӢӨ. м•„лһҳлҠ” мӨ‘ліө л°ңмҶЎмқҙ л°ңмғқн•  мҲҳ мһҲлҠ” мӢңлӮҳлҰ¬мҳӨмҷҖ нҳ„мһ¬ л°©м–ҙ кө¬мЎ°лҘј л¶„м„қн•ң кІ°кіјмһ…лӢҲлӢӨ.

нҳ„мһ¬ лӘЁлҚёмқҳ м„ұкІ©

нҳ„мһ¬ кө¬мЎ°лҠ” exactly-onceк°Җ м•„лӢҢ ліөкө¬ к°ҖлҠҘн•ң at-least-once л°ңмҶЎ лӘЁлҚёмһ…лӢҲлӢӨ.

  • мӨ‘ліө мӢӨн–ү м–өм ңлҠ” мһҳ н•ҳкі  мһҲмқҢ

  • н•ҳм§Җл§Ң мӨ‘ліө л°ңмҶЎ м ҲлҢҖ кёҲм§ҖлҘј мҷёл¶Җ мұ„л„җ мҲҳмӨҖм—җм„ң ліҙмһҘн•ҳм§ҖлҠ” лӘ»н•Ё

к°ҖмһҘ нҒ° мқҙмң лҠ” мҷёл¶Җ л°ңмҶЎ м„ұкіөкіј DB мғҒнғң л°ҳмҳҒ(SENT)мқҙ мӣҗмһҗм Ғмқҙм§Җ м•Ҡкё° л•Ңл¬ёмһ…лӢҲлӢӨ.

мӢңлӮҳлҰ¬мҳӨ 1: мҷёл¶Җ л°ңмҶЎ м„ұкіө нӣ„ нҒ¬лһҳмӢң (мң„н—ҳлҸ„: лҶ’мқҢ)

мғҒнҷ©: Notificationмқҙ мҷёл¶Җ мұ„л„җлЎң мӢӨм ң л°ңмҶЎлҗң м§Ғнӣ„, DBм—җ SENTлЎң л°ҳмҳҒн•ҳкё° м „м—җ н”„лЎңм„ёмҠӨк°Җ мў…лЈҢлҗ©лӢҲлӢӨ.

нқҗлҰ„:

л°©м–ҙ (мӢңмҠӨн…ң л ҲлІЁ): м—ҶмқҢ. Job/Notification мғҒнғң лЁёмӢ м—җм„ңлҠ” л°©м§Җ л¶Ҳк°ҖлҠҘ.

л°©м–ҙ (мұ„л„җ л ҲлІЁ): мҷёл¶Җ л°ңмҶЎ мұ„л„җмқҙ idempotency keyлҘј м§Җмӣҗн•ҳл©ҙ м°ЁлӢЁ к°ҖлҠҘ. мҳҲ: мқҙл©”мқј м„ңл№„мҠӨмқҳ Message-ID, FCMмқҳ send_id.

нҢҗлӢЁ: кө¬нҳ„ мӢӨмҲҳк°Җ м•„лӢҲлқј кө¬мЎ°м Ғ н•ңкі„мһ…лӢҲлӢӨ. мҷёл¶Җ мӢңмҠӨн…ңкіјмқҳ л¶„мӮ° нҠёлһңмһӯм…ҳ м—ҶмқҙлҠ” мӢңмҠӨн…ң л ҲлІЁ мҷ„м „ л°©м§Җ л¶Ҳк°ҖлҠҘ. at-least-once лӘЁлҚёмқҳ к·јліём Ғ н•ңкі„мқҙлҜҖлЎң, мҷёл¶Җ мұ„л„җмқҳ idempotency ліҙмһҘм—җ мқҳмЎҙн•©лӢҲлӢӨ.

мӢңлӮҳлҰ¬мҳӨ 2: к°ҷмқҖ Jobм—җ лҢҖн•ң мқҙмӨ‘ мӢӨн–ү (мң„н—ҳлҸ„: л°©м–ҙлҗЁ)

мғҒнҷ©: `ProcessingNotificationJobExecutionScheduler`к°Җ PROCESSING мғҒнғңмқҳ Jobм—җ лҢҖн•ҙ execution eventлҘј л°ҳліө л°ңн–үн•©лӢҲлӢӨ. л‘җ ExecutionProcessorк°Җ лҸҷмӢңм—җ к°ҷмқҖ Jobмқ„ мІҳлҰ¬н•ҳл Ө н•©лӢҲлӢӨ.

нқҗлҰ„:

л°©м–ҙ: 3лӢЁкі„лЎң л°©м–ҙлҗ©лӢҲлӢӨ.

  1. л¶„мӮ° лқҪ — лҢҖл¶Җл¶„мқҳ мӨ‘ліө м§„мһ…мқ„ м°ЁлӢЁ

  2. SELECT FOR UPDATE — `@ApplicationModuleListener`мқҳ REQUIRES_NEW нҠёлһңмһӯм…ҳ лІ”мң„мҷҖ н–ү мһ кёҲмқҙ мқјм№ҳн•ҳм—¬, л°ңмҶЎ мҷ„лЈҢк№Ңм§Җ лӢӨлҘё мӢӨн–үмһҗк°Җ лҢҖкё°

  3. SENDING м„ м җ м ҖмһҘ — мқҙлҜё SENDING/SENTмқё м•ҢлҰјмқҖ лӢӨмқҢ мӢӨн–үм—җм„ң лҢҖмғҒм—җм„ң м ңмҷё

нҢҗлӢЁ: нҳ„мһ¬ кө¬мЎ°м—җм„ң мқҙ мӢңлӮҳлҰ¬мҳӨлҠ” м•Ҳм „н•©лӢҲлӢӨ.

мӢңлӮҳлҰ¬мҳӨ 3: StuckRecoveryмҷҖ нҷңм„ұ мӢӨн–үмқҳ м¶©лҸҢ (мң„н—ҳлҸ„: л°©м–ҙлҗЁ)

мғҒнҷ©: Jobмқҙ `stuckTimeoutSeconds`лҘј мҙҲкіјн–Ҳм§Җл§Ң мӢӨм ңлЎңлҠ” м•„м§Ғ л°ңмҶЎ мӨ‘мһ…лӢҲлӢӨ. `StuckProcessingRecoveryScheduler`к°Җ мқҙлҘј "мҠӨн„ұ"мңјлЎң мҳӨнҢҗн•©лӢҲлӢӨ.

л°©м–ҙ кө¬мЎ°:

  1. л¶„мӮ° лқҪ лҸҷмқј нӮӨ — ExecutionProcessorмҷҖ StuckRecovery лӘЁл‘җ к°ҷмқҖ idempotencyKey`лЎң лқҪмқ„ мӢңлҸ„. мӢӨн–ү мӨ‘мқҙл©ҙ StuckRecoveryк°Җ `tryLock мӢӨнҢЁлЎң мҠӨнӮө

  2. лқҪ TTL л§ҢлЈҢ мӢң — StuckRecoveryк°Җ лқҪмқ„ нҡҚл“қн•ҙлҸ„ SELECT FOR UPDATEм—җм„ң ExecutionProcessorмқҳ нҠёлһңмһӯм…ҳмқҙ лҒқлӮ  л•Ңк№Ңм§Җ лҢҖкё°. нҠёлһңмһӯм…ҳ мҷ„лЈҢ нӣ„ мғҒнғңлҘј нҷ•мқён•ҳл©ҙ мқҙлҜё COMPLETED/FAILED

нҢҗлӢЁ: мӢңлӮҳлҰ¬мҳӨ 2мҷҖ лҸҷмқјн•ң л©”м»ӨлӢҲмҰҳмңјлЎң л°©м–ҙлҗ©лӢҲлӢӨ.

мӢңлӮҳлҰ¬мҳӨ 4: мҠӨмјҖмӨ„ мӨ‘ліө мІҳлҰ¬ (мң„н—ҳлҸ„: лӮ®мқҢ)

мғҒнҷ©: `DbScheduledNotificationJobRelay`м—җм„ң л‘җ мқёмҠӨн„ҙмҠӨк°Җ к°ҷмқҖ мҠӨмјҖмӨ„ rowлҘј мқҪкі  к°Ғк°Ғ мІҳлҰ¬лҘј мӢңлҸ„н•©лӢҲлӢӨ.

л°©м–ҙ: processScheduledJob`мқҙ REQUIRES_NEW нҠёлһңмһӯм…ҳ м•Ҳм—җм„ң `findByIdAndDeletedFalseForUpdate`лЎң н–ү мһ кёҲмқ„ мһЎмңјлҜҖлЎң, мІ« лІҲм§ё мқёмҠӨн„ҙмҠӨк°Җ SCHEDULED вҶ’ PROCESSING м „мқҙ нӣ„, л‘җ лІҲм§ё мқёмҠӨн„ҙмҠӨлҠ” `markProcessing() мӢӨнҢЁлЎң мҠӨнӮөн•©лӢҲлӢӨ. Job мӨ‘ліө м „мқҙмҷҖ мӨ‘ліө л°ңмҶЎ лӘЁл‘җ л°ңмғқн•ҳм§Җ м•ҠмҠөлӢҲлӢӨ.

мӢңлӮҳлҰ¬мҳӨ 5: Recover UCмҷҖ RetryingScheduler л ҲмқҙмҠӨ (мң„н—ҳлҸ„: м—ҶмқҢ)

мғҒнҷ©: `RecoverNotificationJobUseCase`к°Җ RETRYINGмңјлЎң м „мқҙн•ҳлҠ” лҸҷм•Ҳ `RetryingNotificationJobExecutionScheduler`к°Җ н•ҙлӢ№ Jobмқ„ к°җм§Җн•©лӢҲлӢӨ.

л°©м–ҙ: RecoverUCлҠ” TransactionTemplate м•Ҳм—җм„ң RETRYING м „мқҙ + Notification лҰ¬м…Ӣмқ„ мӣҗмһҗм ҒмңјлЎң мІҳлҰ¬н•ҳкі , л¶„мӮ° лқҪмқ„ нҠёлһңмһӯм…ҳ м»Өл°Ӣ мқҙнӣ„м—җ н•ҙм ңн•©лӢҲлӢӨ. RetryingSchedulerлҠ” к°ҷмқҖ нӮӨлЎң лқҪмқ„ мӢңлҸ„н•ҳлҜҖлЎң UC мҷ„лЈҢ м „к№Ңм§Җ м§„мһ… л¶Ҳк°Җ. мҷ„лЈҢ нӣ„м—җлҠ” Job(RETRYING) + Notification(RETRY_WAITING)мқҙ лӘЁл‘җ м»Өл°Ӣлҗң мғҒнғңмһ…лӢҲлӢӨ.

мӣҗм№ҷ

мҷёл¶Җ м„ңл№„мҠӨ нҳём¶ңмқҙ нҠёлһңмһӯм…ҳ м•Ҳм—җ мһҲмңјл©ҙ л‘җ к°Җм§Җ л¬ём ңк°Җ л°ңмғқн•©лӢҲлӢӨ:

  1. н–ү мһ кёҲ ліҙмң  мӢңк°„ мҰқк°Җ — мҷёл¶Җ нҳём¶ң мқ‘лӢө лҢҖкё° лҸҷм•Ҳ FOR UPDATE мһ кёҲмқҙ мң м§Җлҗҳм–ҙ лӢӨлҘё мӢӨн–ү мЈјмІҙлҘј м°ЁлӢЁ

  2. лЎӨл°ұ мӢң л¶Ҳмқјм№ҳ — мҷёл¶Җ нҳём¶ң м„ұкіө нӣ„ TX лЎӨл°ұ мӢң, мҷёл¶Җм—җлҠ” л°ҳмҳҒлҗҗм§Җл§Ң DBм—җлҠ” л°ҳмҳҒлҗҳм§Җ м•ҠмқҖ мғҒнғң л°ңмғқ

л”°лқјм„ң DB мғҒнғң м „мқҙлҘј лЁјм Җ м»Өл°Ӣн•ҳкі , мҷёл¶Җ нҳём¶ңмқҖ мқҙнӣ„м—җ мҲҳн–үн•©лӢҲлӢӨ.

нҳём¶ң м§Җм җлі„ м„Өкі„

мҷёл¶Җ мҠӨмјҖмӨ„лҹ¬ нҳём¶ңмқҖ `NotificationJobScheduleEvent`лҘј нҶөн•ҙ л¶„лҰ¬лҗ©лӢҲлӢӨ. мқҙлІӨнҠёлҠ” мғҒнғң м „мқҙ TX м•Ҳм—җм„ң л°ңн–үлҗҳкі , Spring Modulithк°Җ TX м»Өл°Ӣ нӣ„ лі„лҸ„ TXм—җм„ң мҶҢл№„н•©лӢҲлӢӨ.

нҠёл Ҳмқҙл“ңмҳӨн”„: мҷёл¶Җ л°ңмҶЎ мқ‘лӢө лҢҖкё° лҸҷм•Ҳ н–ү мһ кёҲмқҙ мң м§Җлҗ©лӢҲлӢӨ. м•ҢлҰј мҲҳк°Җ л§Һкұ°лӮҳ мҷёл¶Җ мұ„л„җ мқ‘лӢөмқҙ лҠҗлҰ¬л©ҙ мһ кёҲ ліҙмң  мӢңк°„мқҙ кёём–ҙм§Ҳ мҲҳ мһҲмҠөлӢҲлӢӨ.
